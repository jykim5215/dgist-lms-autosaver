Google API 발급 도움말

Drive 동기화는 Google OAuth로 권한을 받아 `token.json`을 저장한 뒤 실행됩니다.

  1. Google Cloud Console에서 새 프로젝트를 만들거나 기존 프로젝트를 선택합니다.
  2. API 및 서비스에서 Google Drive API를 사용 설정합니다.
  3. Google Auth Platform에서 앱 이름, 지원 이메일, 사용자 유형을 설정합니다. 개인 테스트라면 테스트 사용자에 본인 Gmail을 추가합니다.
  4. Clients에서 OAuth 클라이언트를 만듭니다. 이 앱은 대시보드에 표시되는 redirect URI를 사용합니다.
  5. 로컬/코드스페이스에서는 credentials.json을 상태 화면의 credentialsPath 위치에 두거나, 배포 서버에서는 환경 변수로 Client ID/Secret을 넣습니다.
  6. 이 화면의 Google OAuth 연결을 누르고 브라우저에서 계정 권한을 허용합니다.
redirect_uri_mismatch가 보이면 대시보드의 Google OAuth 상태에 표시되는 redirect URI와 Google Cloud Console의 Authorized redirect URIs가 같은지 확인해 주세요.
권장 범위 이 앱은 `drive.file` 범위를 사용합니다. 앱이 만든 파일과 사용자가 선택한 파일에 접근하는 비교적 제한적인 권한입니다.
